# HSCI V4 — Testing Standard (TESTING_STANDARD.md)

This document establishes the testing methodologies, coverage requirements, execution frameworks, and gate policies for all code changes in the HSCI repository.

---

## 1. Testing Hierarchy

To verify the reliability of a Cognitive Operating System, tests are categorized into five distinct layers:

```
  ┌────────────────────────────────────────────────────────┐
  │ 1. Unit Tests                                          │
  │ (Mock dependencies, verify isolated function logic)    │
  └───────────────────────────┬────────────────────────────┘
                              │
                              ▼
  ┌────────────────────────────────────────────────────────┐
  │ 2. Integration Tests                                   │
  │ (Verify transactional interactions with UKM SQLite)     │
  └───────────────────────────┬────────────────────────────┘
                              │
                              ▼
  ┌────────────────────────────────────────────────────────┐
  │ 3. Regression Tests                                    │
  │ (Run historical test assertions to check logic drifts) │
  └───────────────────────────┬────────────────────────────┘
                              │
                              ▼
  ┌────────────────────────────────────────────────────────┐
  │ 4. Performance Tests                                   │
  │ (Assert concurrency and memory footprint behaviors)    │
  └───────────────────────────┬────────────────────────────┘
                              │
                              ▼
  ┌────────────────────────────────────────────────────────┐
  │ 5. Benchmark Tests                                     │
  │ (Execute solver accuracy verification suites)           │
  └────────────────────────────────────────────────────────┘
```

---

## 2. Test Layer Specifications

### 2.1 Unit Tests
*   **Scope**: Verify the behavior of single classes or helper functions (e.g., AST syntax whitelist verification in `ConceptCompiler`).
*   **Mocks**: Mock all external resource engines, databases, and Z3 solvers. No database file read/write operations may occur.
*   **Framework**: Use `pytest`.

### 2.2 Integration Tests
*   **Scope**: Verify interactions between cognitive layers and persistence stores (e.g., `LearningEngine` committing updates to SQLite database tables).
*   **Test DB Setup**: Tests must use an isolated database adapter (`InMemoryStore` or a temporary test sqlite file) initialized before and dropped after the run. Never write to the production database during test executions.

### 2.3 Regression Tests
*   **Scope**: Guard against regression bugs. Any time a bug is fixed (such as a GNN classification race condition), a corresponding regression test must be added to the test suite to prevent recurrence.
*   **Assertions**: Verify that prior failure scenarios execute successfully.

### 2.4 Performance Tests
*   **Scope**: Measure execution timing and concurrent resource metrics.
*   **Concurreny Stress Test**: Simulate up to 100 concurrent worker threads sending request payloads to the `BrainKernel` using a pool of database threads. Verify that zero write contention errors or locks occur.
*   **Memory Footprint**: Verify that memory buffers and caches (LRU size 256) maintain bounded footprints.

### 2.5 Benchmark Tests
*   **Scope**: Verify correctness across active solver domains.
*   **Task Suite**: Run the benchmark suite (`tasks.json`).
*   **Threshold**: All five deterministic solvers must maintain a $100\%$ accuracy rate.

---

## 3. Coverage Requirements

*   **Branch Coverage Target**: Branch coverage for any new subsystem or modified component must be $\ge 90\%$.
*   **Enforcement**: Code reviews must inspect coverage statistics generated by `pytest-cov`.
*   **Coverage Calculation**: Skip obsolete or deprecated legacy files from coverage calculations (e.g., legacy scripts under `hnsds/brain/cognitive_core.py`).

---

## 4. Test Execution Protocols

*   **Command Suite**:
    *   Run Unit Tests: `pytest hsci/tests/`
    *   Run Integration Tests: `pytest hsci/tests/ -m integration`
    *   Run Benchmarks: `python hsci_cli.py --run-benchmarks`
*   **cadence**: Tests must be executed before committing changes to the code repository.
